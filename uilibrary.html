Welcome to the LifeasHub UI Library

This document is designed to help you understand all the features and usage of the LifeasHub UI library. With its modern, flexible, and customizable structure, you can easily create stylish interfaces for your projects.

The library is built on a tabbed structure and offers many ready-to-use elements such as buttons, sliders, and dropdown menus. It also comes with advanced features like theme management, a modern notification system, and a flexible configuration system. You can find more info at our official website.

Setup

The library is executed via loadstring. This is the only line you need to run the library and get access to its functions.

local Library = loadstring(game:HttpGet("https://raw.githubusercontent.com/barann222/lifeashub/refs/heads/main/better"))()

Initializing the Library Library:Init

The Library:Init() function is used to create the main window of the interface. This function takes a table containing the basic settings for the window and returns a Window object that you can work with.

local Window = Library:Init({
    name = "PROJECT",
    name2 = "NAME",
    name3 = "v1.0 | GAME NAME",
    folderName = "MyProjectConfigs",
    defaultTheme = "Blurple",
    defaultThemeLevel = 1,
    notificationPlace = "bottom",
    minimizeFrame = {
        enabled = true,
        icon = 'rbxassetid://13529889528'
    }
})

Parameters

ParameterTypeDescription
namestringThe main title in the top-left corner of the window.
name2stringThe second, colored title that appears next to the main title.
name3stringA subtitle that appears to the right of the titles (e.g., version).
folderNamestringThe name of the folder where your configuration files will be saved.
defaultThemestringThe initial theme. E.g., 'Red', 'Green', 'Blurple'.
defaultThemeLevelintegerBackground theme intensity (0: Off, 1: Standard, 2: Intense).
notificationPlacestringSets notification position. Options: 'up' (top-right, default), 'bottom' (bottom-right).
minimizeFrametableA table containing settings to customize the minimized frame.

Minimize Frame Parameters

ParameterTypeDescription
enabledbooleanEnables or disables the custom minimized frame.
iconstringThe asset ID for the icon displayed on the minimized frame.
sizeUDim2The size of the minimized frame.
positionUDim2The on-screen position of the minimized frame.
backgroundColorColor3The background color of the frame.
backgroundTransparencynumberThe transparency of the frame's background (0-1).
imageColorColor3The color of the icon image.

Tabs Window:CreateTab

All interface elements are contained within tabs. You can create a new tab using the Window:CreateTab() function.

local MainTab = Window:CreateTab({ name = 'Main' })
local SettingsTab = Window:CreateTab({ name = 'Settings' })

-- This makes the tab visible when the UI is first opened
MainTab:Activate()

Methods

Tab:Activate(): Makes the corresponding tab visible and active. You should use this method to ensure a tab is displayed when the UI first loads.

Elements

The library provides various elements to build your interface. All elements are created from a Tab object.

Button

A standard button that performs an action when clicked.

Parameters

ParameterTypeDescription
namestringThe text displayed on the button.
callbackfunctionThe function to run when the button is clicked.

Example

MainTab:Button({
    name = "Execute Action",
    callback = function()
        print("Button was clicked!")
        Window:Notify({title="Action", content="The button was executed."})
    end
})

Toggle

A switch that can be turned on or off. Its state is automatically saved by the config system.

Parameters

ParameterTypeDescription
titlestringThe text label for the toggle.
defaultboolean(Optional) The initial state of the toggle (true for on, false for off). Defaults to false.
callbackfunctionThe function to run when the toggle's state changes. It receives the new state as an argument (e.g., function(isEnabled)).

Example

SettingsTab:Toggle({
    title = "Enable Feature",
    default = true, -- Starts in the "on" state
    callback = function(isEnabled)
        print("Feature is now:", isEnabled)
    end
})

Slider

Used to select a numerical value within a specific range. Its value is automatically saved by the config system.

Parameters

ParameterTypeDescription
titlestringThe text label for the slider.
minnumberThe minimum value of the slider.
maxnumberThe maximum value of the slider.
defaultnumber(Optional) The initial value of the slider.
callbackfunctionThe function to run when the slider's value changes. It receives the new value as an argument (e.g., function(value)).

Example

SettingsTab:Slider({
    title = "WalkSpeed",
    min = 16,
    max = 100,
    default = 25,
    callback = function(value)
        print("WalkSpeed set to:", value)
    end
})

Dropdown

A dropdown menu for single or multiple selections. Its selected item(s) are automatically saved by the config system.

Parameters

ParameterTypeDescription
titlestringThe text label for the dropdown.
itemstableA list of strings to populate the dropdown options.
multiSelectboolean(Optional) If true, allows selecting multiple options. Defaults to false.
callbackfunctionThe function to run when a selection is made. If multi-select is off, it receives the selected string. If it's on, it receives a table of selected items.

Examples

-- Single-select example
SettingsTab:Dropdown({
    title = "Target Priority",
    items = {"Closest", "Lowest Health", "Highest Threat"},
    multiSelect = false,
    callback = function(selectedItem)
        print("New priority:", selectedItem)
    end
})

-- Multi-select example
SettingsTab:Dropdown({
    title = "ESP Options",
    items = {"Boxes", "Names", "Tracers", "Health"},
    multiSelect = true,
    callback = function(selectedTable)
        print("ESP options updated:")
        for option, isSelected in pairs(selectedTable) do
            if isSelected then
                print("-", option)
            end
        end
    end
})

TextInput

A box for user text input. Its content is automatically saved by the config system.

Parameters

ParameterTypeDescription
titlestringThe text label for the input box.
placeholderstring(Optional) The placeholder text shown when the box is empty.
callbackfunctionThe function to run when the user presses Enter. It receives the input text as an argument.

Example

MainTab:TextInput({
    title = "Custom Welcome Message",
    placeholder = "Enter a message...",
    callback = function(text)
        Window:Notify({title="Message Set", content = "Your message is: " .. text})
    end
})

Label

Used to separate groups of elements or to add a heading to a section.

Parameters

ParameterTypeDescription
messagestringThe text to display in the label.

Example

SettingsTab:Label({message = "Visual Settings"})

Paragraph

An informational box containing a title, a long description, and an optional image.

Parameters

ParameterTypeDescription
titlestringThe title of the paragraph box.
contentstringThe main text content.
imagestring(Optional) A Roblox asset ID (e.g., rbxassetid://...) for an image to display at the top of the paragraph.

Examples

-- Paragraph without an image
MainTab:Paragraph({
    title = "Welcome!",
    content = "This script has many features. Explore the tabs to find out more."
})

-- Paragraph with an image
MainTab:Paragraph({
    title = "Update Information",
    content = "A new version has been released with major bug fixes and performance improvements.",
    image = "rbxassetid://6019884959" -- Example Image
})

Theme Management

The library includes a robust theme system to change the UI's appearance in real-time. You can control the accent colors and background colors separately.

Set Theme Window:SetTheme

This function changes the primary accent color of the UI elements like highlights, sliders, and dropdowns. The following themes are available:

  • Red
  • Green
  • Dark Green
  • White
  • Black
  • Purple
  • Blurple
-- Example of changing the theme with a dropdown
SettingsTab:Dropdown({
    title = "UI Theme",
    items = {"Red", "Green", "Dark Green", "White", "Black", "Purple", "Blurple"},
    callback = function(selectedTheme)
        Window:SetTheme(selectedTheme)
    end
})

Set Background Theme Level Window:SetBackgroundThemeLevel

This function controls how much the chosen theme affects the background colors of the window and its components.

LevelNameDescription
0OffBackgrounds remain the default dark color, regardless of the theme.
1StandardBackgrounds change to match the selected theme's color palette.
2IntenseBackgrounds are tinted with the theme's main accent color for a more vibrant look.
 
-- Example of controlling the intensity with a slider
SettingsTab:Slider({
    title = "Background Intensity",
    min = 0, max = 2, default = 1,
    callback = function(value)
        local intValue = math.floor(value + 0.5) -- Round to 0, 1, or 2
        Window:SetBackgroundThemeLevel(intValue)
    end
})

Configuration Management

The library provides functions to save, load, and list user configurations. These functions allow you to build your own settings menu with full control.

Save Config Window:SaveConfig

Saves the current state of all Toggles, Sliders, Dropdowns, and TextInputs to a file within the folderName you specified during initialization.

Window:SaveConfig("my_settings") -- Saves settings to 'MyProjectConfigs/my_settings.json'

Load Config Window:LoadConfig

Loads the settings from a specified file and updates the UI elements accordingly.

Window:LoadConfig("my_settings") -- Loads settings from 'MyProjectConfigs/my_settings.json'

Get Configurations Window:GetConfigurations

Returns a table containing the names of all saved configuration files in your specified folder.

local savedConfigs = Window:GetConfigurations()
for i, configName in ipairs(savedConfigs) do
    print("Found config:", configName)
end

Notification System Window:Notify

Displays modern, animated notifications. Position can be set during initialization.

-- Notification with a status icon
Window:Notify({
    title = "Warning!",
    content = "You are about to reset the settings.",
    type = "warning", -- Options: "success", "error", "warning", "info"
    duration = 4
})

-- Notification with a custom banner image
Window:Notify({
    title = "Update Available!",
    content = "Version 2.0 has been released.",
    image = "rbxassetid://2853763354",
    imageStyle = "banner"
})

Full Example

This is a complete example demonstrating how to initialize the library and build a feature-rich UI, including a custom settings menu that uses all theme and configuration functions.

-- 1. Load the library using loadstring
local Library
local success, result = pcall(function()
    Library = loadstring(game:HttpGet("https://raw.githubusercontent.com/barann222/lifeashub/refs/heads/main/better"))()
end)

if not success or not Library or not Library.Init then
    warn("LifeasHub Library failed to load.")
    return
end

-- 2. Initialize the main window using the specified settings
local Window = Library:Init({
    name = "PROJECT",
    name2 = "NAME",
    name3 = "v1.0 | GAME NAME",
    folderName = "MyProjectConfigs",
    defaultTheme = "Blurple",
    defaultThemeLevel = 1,
    notificationPlace = "bottom",
    minimizeFrame = {
        enabled = true,
        icon = 'rbxassetid://13529889528'
    }
})

-- 3. Create all the necessary tabs
local MainTab = Window:CreateTab({ name = "Main" })
local VisualsTab = Window:CreateTab({ name = "Visuals" })
local SettingsTab = Window:CreateTab({ name = "Settings" })

--================================================================================--
-- TAB 1: MAIN - Demonstrates primary action elements
--================================================================================--

MainTab:Paragraph({
    title = "Welcome to the Full Showcase!",
    content = "This UI demonstrates all available features. This tab shows common use-cases like toggles and inputs. The 'Visuals' tab contains ESP and graphics options, and the 'Settings' tab allows you to manage themes and save your configuration.",
    image = "rbxassetid://6019884959"
})

MainTab:Label({ message = "Combat Options" })

MainTab:Toggle({
    title = "Enable Aimbot",
    callback = function(isEnabled)
        print("Aimbot status:", isEnabled)
        Window:Notify({ title = "Aimbot", content = "Aimbot has been " .. (isEnabled and "enabled." or "disabled."), type = "info" })
    end
})

MainTab:Dropdown({
    title = "Aimbot Target Part",
    items = {"Head", "Torso", "HumanoidRootPart"},
    multiSelect = false, -- Single-select dropdown example
    callback = function(selectedPart)
        print("Aimbot target set to:", selectedPart)
    end
})

MainTab:Label({ message = "Player Movement" })

local walkspeedInput = MainTab:TextInput({
    title = "Custom WalkSpeed",
    placeholder = "Enter a number (e.g., 50)...",
    callback = function(text)
        print("Walkspeed input changed to:", text)
    end
})

MainTab:Button({
    name = "Apply WalkSpeed",
    callback = function()
        local speedValue = tonumber(walkspeedInput:GetText())
        if speedValue then
            game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = speedValue
            Window:Notify({ title = "Success", content = "WalkSpeed set to " .. speedValue, type = "success" })
        else
            Window:Notify({ title = "Error", content = "Invalid number for WalkSpeed.", type = "error" })
        end
    end
})


--================================================================================--
-- TAB 2: VISUALS - Demonstrates visual customization elements
--================================================================================--

VisualsTab:Label({ message = "Player ESP Settings" })

VisualsTab:Toggle({
    title = "Enable Player ESP",
    default = true, -- Example of a toggle that starts as ON
    callback = function(isEnabled)
        print("Player ESP status:", isEnabled)
    end
})

VisualsTab:Dropdown({
    title = "ESP Features",
    items = {"Boxes", "Names", "Tracers", "Health Bar"},
    multiSelect = true, -- Multi-select dropdown example
    callback = function(selectedItems)
        print("Current ESP features:")
        local enabledFeatures = {}
        for feature, isSelected in pairs(selectedItems) do
            if isSelected then
                table.insert(enabledFeatures, feature)
            end
        end
        if #enabledFeatures > 0 then
            Window:Notify({ title = "ESP Updated", content = "Enabled: " .. table.concat(enabledFeatures, ", ") })
        else
            Window:Notify({ title = "ESP Updated", content = "All ESP features are disabled.", type = "warning" })
        end
    end
})

VisualsTab:Label({ message = "World & View" })

VisualsTab:Slider({
    title = "Field of View (FOV)",
    min = 70,
    max = 120,
    default = 80,
    callback = function(value)
        workspace.CurrentCamera.FieldOfView = value
    end
})


--================================================================================--
-- TAB 3: SETTINGS - Demonstrates theme and config management
--================================================================================--

SettingsTab:Label({ message = "Theme Controls" })

SettingsTab:Dropdown({
    title = "UI Theme",
    items = {"Red", "Green", "Dark Green", "White", "Black", "Purple", "Blurple"},
    multiSelect = false,
    callback = function(selectedTheme)
        Window:SetTheme(selectedTheme)
    end
})

SettingsTab:Slider({
    title = "Background Intensity",
    min = 0,
    max = 2,
    default = 1,
    callback = function(value)
        local intValue = math.floor(value + 0.5)
        Window:SetBackgroundThemeLevel(intValue)
    end
})

SettingsTab:Label({ message = "Configuration Management" })

local ConfigsDropdown = SettingsTab:Dropdown({
    title = "Available Configs",
    items = {},
    callback = function(selectedConfig)
        Window:LoadConfig(selectedConfig)
    end
})

local refreshButton = SettingsTab:Button({
    name = "Refresh Configs",
    callback = function()
        ConfigsDropdown:Clear()
        local configFiles = Window:GetConfigurations()
        for _, configName in ipairs(configFiles) do
            ConfigsDropdown:Add(configName, configName)
        end
        Window:Notify({title="Refreshed", content= #configFiles .. " configs found.", type="info"})
    end
})

local NewConfigInput = SettingsTab:TextInput({
    title = "New Config Name",
    placeholder = "E.g: pvp_settings"
})

SettingsTab:Button({
    name = "Save Current Config",
    callback = function()
        local configName = NewConfigInput:GetText()
        if configName and #configName > 0 then
            Window:SaveConfig(configName)
            if refreshButton then
                refreshButton.options.callback()
            end
        else
            Window:Notify({title = "Warning", content = "Please enter a valid file name.", type = "warning"})
        end
    end
})


-- 4. Activate the main tab to make it visible on start
MainTab:Activate()

-- 5. Initially populate the config list by triggering the refresh function
if refreshButton then
    refreshButton.options.callback()
end
